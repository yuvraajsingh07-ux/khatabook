🧾 Replit Vibe-Coding Prompt — Vendor/Customer Payment Ledger App

📌 What I'm Building
A mobile-first React web app for tracking payments between me and multiple vendors/customers. Each party has their own ledger (cash book) — a running log of bills and payments with a live balance. Think of it as a per-party digital khata/passbook.

🎨 Design Language

Dark mode only — deep background (#0f1117 or similar), card surfaces at #1a1d27, accent color a clean indigo or teal (#6366f1 or #14b8a6)
Card-based layout — every entry is a soft card with subtle border/glow, no flat table rows
Typography: Inter or DM Sans, large numeric fonts for amounts
Consistent 16px border-radius on cards, modals, buttons
Smooth transitions and micro-animations on interactions
Color system: green = credit/cash in, red = debit/bill, blue = transfer, amber = pending


🏠 Ledger List Screen (Home)

Shows all ledgers as cards in a scrollable list
Each ledger card displays:

Party name (large)
Pending balance shown prominently on the right side of the card — green if they owe me, red if I owe them
A small FIFO badge if FIFO mode is on for that ledger (e.g. a small amber chip showing oldest unpaid bill balance)
Tap → opens that ledger's transaction screen


"+ Add New Ledger" button at the bottom
Each ledger card has a 3-dot menu → options: Open, View Profile, Rename, Delete


👤 Ledger Profile

Every ledger has a Profile screen accessible from the ledger card or inside the ledger view
Profile stores:

Party name
Phone number
UPI ID
Bank account number + IFSC
Any notes/remarks about the party


This screen is read/edit — tap a field to edit it inline
A "Copy UPI" and "Pay via UPI" quick-action button on the profile card


📒 Ledger Detail Screen (Per Party)

Top bar:

Party name (tappable → opens party switcher as bottom sheet)
Current net balance shown large just below name (green/red)
Right side: profile icon → opens Ledger Profile, settings/gear icon → options menu


Sticky summary strip below top bar:

Total Credit | Total Debit | Net Balance for visible period


Entries grouped by date — date shown as a centered pill chip: ── 27 Mar 2026 ──
Each entry card shows:

Remark/Particular text (wraps fully, no truncation)
Amount large font (green = credit, red = debit, blue = transfer)
Running balance below in small grey
A small type tag: BILL / PAYMENT / TRANSFER


Swipe left on entry → Edit / Delete


➕ Add Entry — Bottom Sheet
Triggered by a FAB (floating action button) in the bottom right — expands into 3 action pills on tap:
1. 💰 Cash In (Credit)

Someone paid me — reduces what they owe

2. 🧾 Bill Out (Debit)

I gave goods on credit — adds to what they owe

3. 🔁 Transfer

Move money between two ledgers — e.g. if Party A's payment is being forwarded to Party B
Fields: Amount, From Ledger (auto-set to current), To Ledger (dropdown/search), Date, Remark
Creates a credit entry in current ledger and a debit entry in the target ledger simultaneously, linked by a transfer ID
Entry shows as blue in both ledgers with a 🔁 icon

All entry forms (Cash In / Bill Out / Transfer) share this structure:

Type selector (segmented pill at top, color-highlighted)
Date (tap → native date picker, defaults to today)
Amount (large numeric input, numeric keyboard auto-opens)
Remark (text input with autocomplete from past entries for this ledger)
Save button — full width, colored by type


📊 FIFO Mode (Per Ledger Toggle)

Each ledger has a FIFO ON/OFF toggle in the ledger settings or top bar
What it does: In this business, goods are sold on credit (udhaar) and customers pay randomly without specifying which bill they're clearing. FIFO mode tracks payments against the oldest unpaid bills first.
When FIFO is ON:

Each bill entry is tracked individually with its own remaining amount
Payments are auto-applied to the oldest bill first, then spill over to the next
In the ledger view, each bill entry shows a small progress bar or "₹X remaining" tag in amber
A FIFO Summary card pinned at the top of the ledger shows:

Oldest unpaid bill date
Amount originally billed
Amount paid against it so far
Amount still pending on that bill


Bills that are fully paid collapse/fade to show they're settled


When FIFO is OFF — normal running balance mode (current behavior)


☁️ Cloud Storage

Firebase Firestore as the backend — all data synced in real-time
Firebase Auth — simple phone number OTP login (no email/password needed, fits the user profile)
Data is always available across devices — no local-only mode
Offline support via Firestore's built-in offline persistence (works without internet, syncs when back online)
Each user's data is scoped to their UID — fully private


📄 PDF Export

Table format: Date | Particular | Credit | Debit | Balance
Header: Party name, date range, generated date
Footer summary: Total Credit, Total Debit, Closing Balance
Opening balance row at top
Triggered from the options/settings icon inside the ledger view
